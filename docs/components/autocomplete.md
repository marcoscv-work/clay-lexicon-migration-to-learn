# Autocomplete

Use an autocomplete when the user types into a field and should get matching
suggestions from a known set: long lists where a plain
[select](/components/select) would be unwieldy, or lookups against server data.

<!-- REVIEW: Lexicon has no standalone autocomplete spec; the behavior appears inside the Multi Select spec. Design guidance here is derived from the Clay implementation and related Lexicon patterns; confirm with the design team. -->

## Variants and anatomy

The field filters its option list as the user types, highlighting the matching
fragment. Options can be static, dynamic from an array, or loaded
asynchronously; a loading indicator shows while results arrive.

Try the [Autocomplete story](https://storybook.clayui.com/?path=/story/design-system-components-autocomplete--default),
the [async data story](https://storybook.clayui.com/?path=/story/design-system-components-autocomplete--async-data),
and the [keyboard story](https://storybook.clayui.com/?path=/story/design-system-components-autocomplete--keyboard).

## Usage guidelines

- Use autocomplete when users know roughly what they are looking for; keep a
  browsable control for exploratory choices.
- Debounce server lookups and show the loading state.
- Allow creating a new value only when the use case calls for it, and make the
  creation option explicit in the list.

## Implementation

Install the component and the CSS package:

```shell
npm install @clayui/css @clayui/autocomplete
```

Provide the options as `items`; the component filters as the user types:

```jsx
import Autocomplete from '@clayui/autocomplete';

export default function Example() {
	return (
		<Autocomplete
			aria-labelledby="fruitLabel"
			id="fruit"
			items={['Apple', 'Banana', 'Blueberry', 'Cherry']}
			messages={{loading: 'Loading...', notFound: 'No results found'}}
			placeholder="Enter a fruit"
		>
			{(item) => <Autocomplete.Item key={item}>{item}</Autocomplete.Item>}
		</Autocomplete>
	);
}
```

## Accessibility

- The component implements the ARIA combobox pattern: the list is announced,
  arrow keys move through suggestions, and `Enter` accepts one.
- Label the field (`aria-labelledby` or a visible label).
- The "no results" and loading messages are announced through the messages you
  provide.
