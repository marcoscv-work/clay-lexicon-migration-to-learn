# Badge

Use a badge to highlight a short numeric value, such as a count of notifications
or unread messages. Badges are for integers only, and they draw attention to a
number without interrupting the user.

## Variants and anatomy

Badges come in several colors for different contexts. A badge has rounded corners
and shows a single value. Try the colors live in the
[Badge story](https://storybook.clayui.com/?path=/story/design-system-components-badge--default),
and the icon variant in the
[badge with icon story](https://storybook.clayui.com/?path=/story/design-system-components-badge--with-icon).

Reserve the red (danger) badge for two cases:

- **Notifications**, placed near a user profile or a product menu option.
- **Errors**, shown in tables and lists.

## Usage guidelines

- Use badges for integers only. For non-numeric values, use a label instead.
- Do not use a badge as a close button or a link. A badge is informative text.
- Show exact numbers up to 999. Above that, abbreviate: use K for thousands (5K)
  and M for millions (2M).
- When the exact count does not matter, use plain text instead of a badge.
- Keep at least 8px of space between a badge and any adjacent element.

## Implementation

Install the component and the CSS package:

```shell
npm install @clayui/css @clayui/badge
```

Set the color with `displayType` and the value with `label`:

```jsx
import Badge from '@clayui/badge';

export default function Example() {
	return <Badge displayType="primary" label="100" />;
}
```

Add an optional icon with `symbol`, or use `translucent` for a lighter
background:

```jsx
import Badge from '@clayui/badge';

export default function Example() {
	return (
		<>
			<Badge displayType="success" label="100" symbol="check" />
			<Badge displayType="info" label="Info" translucent />
		</>
	);
}
```

## Accessibility

- A badge on its own communicates meaning through a number, not color, so a value
  such as "100" is understandable without seeing the color.
- When a badge represents a count for a nearby control (for example unread
  messages on a menu item), make sure the control's accessible name includes the
  count or its meaning, so screen reader users get the same information.
- Avoid icon only badges without a `label`, since an icon alone does not convey a
  value.
