# Breadcrumb

Use a breadcrumb as secondary navigation that identifies the current page's
position inside a hierarchy, for example while navigating folders in a table or
list. Every crumb links to its level, except the last one, which only identifies
the current page.

## Variants and anatomy

The breadcrumb is always horizontal and works the same across viewport sizes.
When there is not enough room to show every level, the first levels collapse
into a drop down menu, with the items in reverse order, so the user can still
reach any level. Long crumbs can be shortened with an ellipsis.

Try the [Breadcrumb story](https://storybook.clayui.com/?path=/story/design-system-components-breadcrumb--default)
and the [active state story](https://storybook.clayui.com/?path=/story/design-system-components-breadcrumb--active-state).

## Usage guidelines

- Use a breadcrumb only for hierarchy. Do not use it for navigation across
  applications.
- Do not show a breadcrumb when there is only one navigation level, and do not
  use it on the first level.
- Keep the breadcrumb to a single line of text.

## Implementation

Install the component and the CSS package:

```shell
npm install @clayui/css @clayui/breadcrumb
```

Pass the trail as `items`; the active item renders as plain text and the rest as
links. Items beyond `ellipsisBuffer` collapse into the drop down:

```jsx
import Breadcrumb from '@clayui/breadcrumb';

export default function Example() {
	return (
		<Breadcrumb
			ellipsisBuffer={1}
			items={[
				{href: '/documents', label: 'Documents'},
				{href: '/documents/reports', label: 'Reports'},
				{active: true, label: '2026'},
			]}
		/>
	);
}
```

## Accessibility

- The breadcrumb renders as a navigation landmark with an ordered list, so
  assistive technology announces the trail structure.
- The current page (the active item) is not a link, which prevents a confusing
  self-referencing link.
- The collapsed levels stay reachable by keyboard through the ellipsis drop
  down.
