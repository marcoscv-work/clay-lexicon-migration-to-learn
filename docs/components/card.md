# Card

Use cards to present a collection of items that are centered on imagery, such as
a library of images or documents, or a directory of people. When the content has
no associated images and needs exhaustive comparison, use a [table](/components/table)
instead.

## Variants and anatomy

### Anatomy

![A card with its parts labeled: checkbox, image, sticker, title, actions button, description text, and status label](/assets/components/card-anatomy.png)

A card can include a selection control (checkbox or radio), an image, a sticker
(for example a file type), a title, an actions button, descriptive text, and a
status label. The image is the only required part; everything else is optional.

### States

A card has default, hover, and selected states, plus an empty state when there is
no image. The hover and selected states use a colored border, and the selected
state also shows a check.

### Variation types

- **Image card**: a photo with a title and description.
- **User card**: a profile image, or initials when there is no photo.

  ![An image card showing a photo, a title, and a short description](/assets/components/card-image.png)

- **File card**: a file type icon centered in place of a photo.
- **Folder card**: a horizontal card with no image and a selection checkbox.

  ![A folder card in a horizontal layout with a selection checkbox and no image](/assets/components/card-folder.png)

- **Navigation card**: a minimal card with an icon or image, a title, and a short
  description, available in big and small formats.
- **Product card**: a product image on a colored background with a call to
  action.

  ![A product card with a product image on a colored background and an add to cart action](/assets/components/card-product.png)

## Usage guidelines

- Use cards only when the content has associated images. For dense, image free
  comparison, use a table.
- Make the card title the main navigation: a folder card opens the next level, and
  a file, image, or user card opens the detail view.
- Show a tooltip with the full title when the title is truncated.
- Visually flag a link that opens in a new page or leaves the site.
- Do not use card navigation for previews; offer a preview as a secondary action
  in the actions menu.

## Implementation

Install the component and the CSS package:

```shell
npm install @clayui/css @clayui/card
```

Compose a card from the low level parts when you need full control:

```jsx
import Card from '@clayui/card';
import Button from '@clayui/button';

export default function Example() {
	return (
		<Card>
			<Card.Body>
				<Card.Description displayType="title">Card title</Card.Description>
				<Card.Description displayType="text" truncate={false}>
					A short description that supports the title.
				</Card.Description>
				<Button>Go somewhere</Button>
			</Card.Body>
		</Card>
	);
}
```

For the common Lexicon cases, use a high level component such as
`ClayCardWithInfo` (for images and documents), `ClayCardWithUser`,
`ClayCardWithHorizontal` (folders), or `ClayCardWithNavigation`:

```jsx
import {ClayCardWithInfo} from '@clayui/card';

export default function Example({selected, onSelectChange}) {
	return (
		<ClayCardWithInfo
			description="A cool description"
			href="#"
			labels={[{displayType: 'success', value: 'Awesome'}]}
			onSelectChange={onSelectChange}
			selected={selected}
			stickerProps={{content: 'DOC', displayType: 'danger'}}
			title="Selectable file"
		/>
	);
}
```

## Accessibility

- Provide alternative text for a card's image through the image props, so the
  image is not announced as unlabeled.
- When a title is truncated, expose the full title (for example through a tooltip)
  so the information is not lost.
- Make links that leave the site or open a new page identifiable beyond color, so
  the behavior is clear to everyone.
