# Checkbox, Radio, and Toggle

These three controls cover selection and activation in forms. Choose by
behavior: checkboxes for selecting several options, radios for selecting exactly
one, and toggles for switching a setting on or off with immediate effect.

## Variants and anatomy

### Checkbox

A checkbox selects the value written in its label, and several can be selected
at once. States: on, off, disabled (both), and indeterminate for partial
multiple selection. Clay renders the native checkbox, which users recognize
across browsers. Try the
[Checkbox story](https://storybook.clayui.com/?path=/story/design-system-components-checkbox--default).

### Radio

A radio selects exactly one value from a group. States: on, off, and their
disabled variants. Group radios in forms; for long lists, one column reads
fastest, and only split into columns when you have at least six options per
column. Try the
[Radio story](https://storybook.clayui.com/?path=/story/design-system-components-radio--default).

### Toggle

A toggle switches one setting on or off and applies immediately, without a
separate save action. Try the
[Toggle story](https://storybook.clayui.com/?path=/story/design-system-components-toggle--checkbox).

## Usage guidelines

- Capitalize the control's label.
- Use checkboxes when more than one option can be selected; use radios when
  only one can.
- Do not use a list of toggles where checkboxes belong: toggles are for
  settings that take effect immediately, not for selecting options in a form.

## Implementation

The three controls ship in `@clayui/form`:

```shell
npm install @clayui/css @clayui/form
```

```jsx
import {ClayCheckbox, ClayRadio, ClayRadioGroup, ClayToggle} from '@clayui/form';
import {useState} from 'react';

export default function Example() {
	const [checked, setChecked] = useState(false);
	const [option, setOption] = useState('one');
	const [on, setOn] = useState(false);

	return (
		<>
			<ClayCheckbox checked={checked} label="Subscribe" onChange={() => setChecked(!checked)} />
			<ClayRadioGroup onChange={setOption} value={option}>
				<ClayRadio label="Option one" value="one" />
				<ClayRadio label="Option two" value="two" />
			</ClayRadioGroup>
			<ClayToggle label="Enabled" onToggle={setOn} toggled={on} />
		</>
	);
}
```

## Accessibility

- Every control needs a label; Clay's `label` property renders and associates
  it.
- Group radios with a group label so the set is announced together
  (`ClayRadioGroup` handles the grouping semantics).
- The indeterminate checkbox state is exposed to assistive technology; use it
  only to summarize a partial selection, with the real options listed below.
