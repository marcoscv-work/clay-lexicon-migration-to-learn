# Data Provider

Use the data provider to fetch remote data for a component, with caching,
polling, and network state handled for you. It powers dynamic content in
components such as [Autocomplete](/components/autocomplete) and
[Multi Select](/components/multi-select), and is available directly as a
component and as the `useResource` hook.

<!-- REVIEW: Clay-only utility (no Lexicon spec). -->

## Variants and anatomy

The provider takes a `link` (URL or fetch function) and renders through a
children function that receives the resource and the network status. Options
cover polling intervals, variables that re-trigger the request, and a custom
fetcher.

Try the [polling story](https://storybook.clayui.com/?path=/story/design-system-components-dataprovider--polling-example)
and the [custom fetcher story](https://storybook.clayui.com/?path=/story/design-system-components-dataprovider--custom-fetcher).

## Usage guidelines

- Show a [loading indicator](/components/loading-indicator) while the first
  load resolves, and an [empty state](/components/empty-state) when the result
  has no items.
- Use polling only for data that genuinely changes; prefer explicit refresh
  otherwise.

## Implementation

Install the package:

```shell
npm install @clayui/data-provider
```

```jsx
import {useResource} from '@clayui/data-provider';

export default function Example() {
	const {resource} = useResource({link: 'https://example.com/api/items'});

	return <pre>{JSON.stringify(resource, null, 2)}</pre>;
}
```

### DataProvider

| Prop | Type | Default | Required | Description |
| ---- | ---- | ------- | -------- | ----------- |
| `children` | `(props: ChildrenProps) => React.ReactElement` |  | Yes | It uses a render props pattern made popular by libraries like React Motion and React Router. Children as a function is required for the DataProvider to pass the props with data information, network status, refetch method and others. If this is an impediment try using the `useResource` hook. |
| `notifyOnNetworkStatusChange` | `boolean` | `false` | No | Set to true means that network status information will be passed via `renders props` and will also cause new renderings as networkStatus changes, when false rendering does not happen again. |

## Accessibility

- Announce loading and error states through the consuming component (live
  region or status text); the provider itself renders nothing.
