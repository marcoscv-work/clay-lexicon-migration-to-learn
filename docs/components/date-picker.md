# Date Picker

Use the date picker to let users select a date for a form from a calendar
panel. It targets desktop browsers; on mobile devices, use the operating
system's native control instead.

## Variants and anatomy

The calendar panel combines: a month selector, a year selector (keep past and
future years available as the use case needs), back and next arrows that roll
over between years, a dot button that jumps to the current date, and the days
panel, where day numbers are interactive and weekday labels are not.

Calendar days respond with default, hover, active (selected, highlighted with a
circle in the primary color), and disabled states.

Variants to try live:

- The default [Date Picker story](https://storybook.clayui.com/?path=/story/design-system-components-datepicker--default).
- [Date and time together](https://storybook.clayui.com/?path=/story/design-system-components-datepicker--time),
  when the form needs both.
- [A date range](https://storybook.clayui.com/?path=/story/design-system-components-datepicker--date-range).
- [The native variant](https://storybook.clayui.com/?path=/story/design-system-components-datepicker--native),
  which delegates to the browser control.

An extension area is available at the end of the calendar panel. If you need a
double calendar, place the two calendars side by side, not stacked.

## Usage guidelines

- Let the user type the date as well as pick it; the field and the calendar
  stay in sync.
- Disable dates that are not valid instead of validating after the fact.
- Use the range variant for periods rather than two disconnected fields.

## Implementation

Install the component and the CSS package:

```shell
npm install @clayui/css @clayui/date-picker
```

Control the value and configure the years the selector offers:

```jsx
import DatePicker from '@clayui/date-picker';
import {useState} from 'react';

export default function Example() {
	const [value, setValue] = useState('');

	return (
		<DatePicker
			onChange={setValue}
			placeholder="YYYY-MM-DD"
			value={value}
			years={{end: 2030, start: 2020}}
		/>
	);
}
```

Add `time` to include the time picker in the same panel.

## Accessibility

- The field works as a regular labeled input: users can type the date without
  opening the calendar.
- The calendar is keyboard operable: arrows move between days, and month and
  year selectors are regular controls.
- Weekday headers are not interactive and are exposed as column headers of the
  grid.
