# Drop Down

Use a drop down to show a list of options or actions for the element that
triggers it. A drop down keeps the screen clean by condensing options until the
user needs them.

## Variants and anatomy

A button or an icon button triggers the drop down, and a caret icon signals the
behavior. The menu opens with an 8px gap from its trigger.

![A default drop down menu open below its trigger button, showing a list of options](/assets/components/drop-down-default.png)

Keep the menu within its size limits: up to 240px wide (wrap long text to two
lines) and up to 500px tall on desktop or 400px on mobile, with vertical
scrolling beyond that.

Common types:

- **Default**: a simple list of options.
- **With dividers**: related actions separated into groups.
- **With checkboxes or radios**: checkboxes for filtering, radios for ordering.

  ![A drop down menu containing a list of checkboxes](/assets/components/drop-down-checkbox.png)

- **With a search field**: filter a long list.

  ![A drop down menu with a search field at the top for filtering options](/assets/components/drop-down-search.png)

- **With groups**: grouped filter and order controls, for example in a management
  bar.

  ![A drop down with grouped filter and order controls and primary and secondary buttons](/assets/components/drop-down-groups.png)

## Usage guidelines

- Close the menu when the user selects a mutually exclusive option, clicks the
  trigger again, or clicks outside.
- Order an actions menu by context: group related actions with separators, keep
  each group to 7 items or fewer, and keep the whole menu to 15 items or fewer.
  Order groups by how often they are used, and order items alphabetically within
  a group.
- Show only the actions that are relevant in context. Disable an action that is
  temporarily unavailable rather than removing it, unless the item can never
  perform that action.
- Do not add submenus or otherwise increase the menu's complexity. For multi
  level navigation, use a drilldown menu.

## Implementation

Install the component and the CSS package:

```shell
npm install @clayui/css @clayui/drop-down
```

Compose the menu from low level parts for full control:

```jsx
import DropDown from '@clayui/drop-down';
import Button from '@clayui/button';

export default function Example() {
	return (
		<DropDown trigger={<Button>Actions</Button>}>
			<DropDown.ItemList>
				<DropDown.Item>Edit</DropDown.Item>
				<DropDown.Item>Duplicate</DropDown.Item>
				<DropDown.Item>Delete</DropDown.Item>
			</DropDown.ItemList>
		</DropDown>
	);
}
```

For a data driven menu, pass `items` and render each one, and add
`DropDown.Search` for out of the box filtering. For property driven menus, use the
high level `ClayDropDownWithItems`, or `ClayDropDownWithDrilldown` for multi level
navigation:

```jsx
import {ClayDropDownWithItems} from '@clayui/drop-down';
import Button from '@clayui/button';

export default function Example() {
	return (
		<ClayDropDownWithItems
			items={[
				{href: '#', title: 'Clickable'},
				{type: 'divider'},
				{href: '#', title: 'Linkable'},
			]}
			trigger={<Button>Actions</Button>}
			triggerIcon="caret-bottom"
		/>
	);
}
```

## Accessibility

- Prefer composing the drop down or using the high level components. They provide
  focus management out of the box: focus moves into the menu when it opens and
  returns to the trigger when it closes.
- The bare low level `ClayDropDown.Menu` does not provide focus control or other
  accessibility behavior on its own. Use it only as a last resort, when the
  trigger lives outside the same component tree.
- Keep drop downs keyboard operable: users open the menu, move through items with
  the arrow keys, and select with `Enter`.
