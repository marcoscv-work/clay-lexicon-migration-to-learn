# Dual Listbox

Use a dual listbox to compare and move options between two lists: what is in use
and what is available. Users can select several items at once, and the in-use
list can optionally allow manual ordering.

## Variants and anatomy

Four parts make the pattern:

- **In Use listbox** (left): the selected options, optionally orderable.
- **Available listbox** (right): the fixed set of remaining options.
- **Transfer buttons**: always visible carets between the lists; their enabled
  state guides the user.
- **Order buttons**: only shown when the in-use list allows manual ordering and
  has more than one item.

Transfer behavior: buttons stay disabled until items are selected; moved items
stay highlighted in the in-use list as confirmation. With manual ordering, new
items land at the bottom; without it, items sort alphabetically. Items moved
back to the available list always sort alphabetically.

Try the [Dual Listbox story](https://storybook.clayui.com/?path=/story/design-system-components-duallistbox--default).

## Usage guidelines

- Label both boxes with exclusive titles ("Selected" and "Not selected", "East
  coast teams" and "West coast teams").
- Add a short task description above the component, so the goal of the transfer
  is clear.
- Keep scrolling available in long lists.

## Implementation

The component ships in `@clayui/form`:

```shell
npm install @clayui/css @clayui/form
```

Provide both lists as a two element array and control the selection state:

```jsx
import {ClayDualListBox} from '@clayui/form';
import {useState} from 'react';

export default function Example() {
	const [items, setItems] = useState([
		[
			{label: 'Selected', options: [{label: 'Editor', value: 'editor'}]},
		],
		[
			{label: 'Available', options: [{label: 'Viewer', value: 'viewer'}]},
		],
	]);

	return (
		<ClayDualListBox
			items={items}
			left={{label: 'In Use'}}
			onItemsChange={setItems}
			right={{label: 'Available'}}
		/>
	);
}
```

## Accessibility

- Both listboxes are labeled, and multiple selection works with the keyboard
  (arrows plus modifier keys).
- Transfer and order buttons have accessible names describing their direction.
- Disabled buttons communicate why nothing happens before a selection exists.
