# Empty State

Use an empty state to explain why a view has nothing to show and what the user
can do to move out of it. A good empty state turns a dead end into a next step.

## Variants and anatomy

### Scenarios

Three default animations cover the common scenarios:

- **Empty dataset**: there are no elements at this level yet. Try the
  [empty state story](https://storybook.clayui.com/?path=/story/design-system-components-emptystate--empty-state).
- **Empty search results**: the filters or search returned nothing. Try the
  [search state story](https://storybook.clayui.com/?path=/story/design-system-components-emptystate--search-state).
- **Success**: the user emptied the dataset for a good reason, for example all
  notifications were handled. Try the
  [success state story](https://storybook.clayui.com/?path=/story/design-system-components-emptystate--success-state).

### Anatomy

An empty state combines up to four parts: the animation or image, a one line
title (required), a message, and a primary or secondary button for the next
action. The margin from the container to the top element is 80px.

### Secondary empty state

An animation is not always necessary. The secondary composition uses only the
title, the message, and a small button.

## Usage guidelines

Write the message carefully; it is doing the explaining:

- Keep the title to one line and the message to about 30 words.
- Use common language, not jargon.
- Do not make the message personal, do not use "your", and do not use
  exclamation marks.
- Do not leave a single word alone on a line.
- Offer a way out when one exists: a button to create the first item or to
  clear the filters.

## Implementation

Install the component and the CSS package:

```shell
npm install @clayui/css @clayui/empty-state
```

Without `imgSrc` you get the default composition; add an image or animation
with `imgSrc`, and pass children for the action:

```jsx
import EmptyState from '@clayui/empty-state';
import Button from '@clayui/button';

export default function Example() {
	return (
		<EmptyState
			description="Create your first entry to see it listed here."
			imgProps={{alt: 'A satellite drifting in space'}}
			imgSrc="/images/empty_state.svg"
			imgSrcReducedMotion="/images/empty_state_reduced_motion.svg"
			title="No entries yet"
		>
			<Button displayType="primary">New Entry</Button>
		</EmptyState>
	);
}
```

## Accessibility

- Give the image meaningful `alt` text through `imgProps`, or empty `alt` if it
  is purely decorative.
- Provide `imgSrcReducedMotion` so users who minimize non essential motion get
  a static image; the `c-prefers-reduced-motion` class forces it for a whole
  container.
- Keep the explanation in text (title and message), never only in the
  illustration.
