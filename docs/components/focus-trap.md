# Focus Trap

Use the focus trap to keep keyboard focus cycling inside a container while it
is active, the behavior that overlays such as [modals](/components/modal) need.
Clay's modal and drop down already trap focus; reach for this utility when you
build a custom overlay.

<!-- REVIEW: Clay-only utility component (no Lexicon spec). -->

## Variants and anatomy

The trap wraps its children: `Tab` from the last focusable element wraps to the
first, and `Shift+Tab` from the first wraps to the last. It can focus a
specific element on activation.

Try the [Focus Trap story](https://storybook.clayui.com/?path=/story/design-system-components-focustrap--default)
and the [focus on specific element story](https://storybook.clayui.com/?path=/story/design-system-components-focustrap--focus-on-specific-element).

## Usage guidelines

- Trap focus only while an overlay is open, and release it when it closes.
- Return focus to the trigger element on release.
- Never trap focus in content that is not visually isolated from the page.

## Implementation

The component ships in `@clayui/core`:

```shell
npm install @clayui/css @clayui/core
```

```jsx
import {FocusTrap} from '@clayui/core';
import Button from '@clayui/button';

export default function Example({active}) {
	return (
		<FocusTrap active={active}>
			<div className="sheet">
				<input placeholder="Name" type="text" />
				<Button>Save</Button>
			</div>
		</FocusTrap>
	);
}
```

### FocusTrap

| Prop | Type | Default | Required | Description |
| ---- | ---- | ------- | -------- | ----------- |
| `active` | `boolean` | `false` | No | Flag to indicate if the focus trap is activated. |
| `children` | `React.ReactNode` |  | Yes | The elements that will receive the focus within the focus trap. |
| `focusElementRef` | `React.RefObject<HTMLElement>` |  | No | Ref of the element that receives the focus when the focus trap is activated. |
| `stopPropagation` | `boolean` | `false` | No | Flag that indicates whether the FocusScope stops keyboard event propagation to parent FocusScope components. |

## Accessibility

- A focus trap is itself an accessibility tool; misused, it becomes a barrier.
  Always pair it with a visible overlay and an `Esc` path out.
- Ensure at least one focusable element exists inside the trap.
