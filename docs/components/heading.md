# Heading

Use the heading component to render titles that follow the
[typography foundation](/foundations/typography): the type scale levels with
their weights and line heights, while keeping the HTML heading level you need
for the document outline.

## Variants and anatomy

The `level` sets both the rendered element (h1 to h6) and the default size; the
`fontSize` can diverge from the level when the visual scale and the document
outline must differ. Weight variants follow the foundation (regular, semibold,
bold).

Try the [heading typography story](https://storybook.clayui.com/?path=/story/design-system-components-typography--heading-typography).

## Usage guidelines

- Choose the level for the outline (one h1 per page, no skipped levels), and
  adjust `fontSize` if the design needs a different visual size.
- Follow the type styles from the foundation for each context (page title, form
  title, section).

## Implementation

The component ships in `@clayui/core`:

```shell
npm install @clayui/css @clayui/core
```

```jsx
import {Heading} from '@clayui/core';

export default function Example() {
	return (
		<Heading fontSize={5} level={2} weight="semi-bold">
			Site Settings
		</Heading>
	);
}
```

### Heading

| Prop | Type | Default | Required | Description |
| ---- | ---- | ------- | -------- | ----------- |
| `children` | `React.ReactNode` |  | No | Heading content. |

## Accessibility

- The heading level defines the document outline that assistive technology
  navigates; never pick a level for its size alone (use `fontSize` for that).
