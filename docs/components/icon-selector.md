# Icon Selector

Use the icon selector to let users pick an icon from the Lexicon icon library,
for example when configuring a navigation entry or a custom action. It combines
a trigger with a searchable panel of icons.

<!-- REVIEW: Clay-only component (no standalone Lexicon spec); the icons foundation rules apply. -->

## Variants and anatomy

The panel lists the spritemap icons in a grid with a search field to filter
them by name. The selection is reflected in the trigger.

Try the [Icon Selector story](https://storybook.clayui.com/?path=/story/design-system-components-iconselector--default).

## Usage guidelines

- Offer only the icons that make sense in the context, if the full library
  would overwhelm the choice.
- Show the icon's name along with the glyph; names carry the meaning.
- Pair the selector with a preview of where the icon will appear.

## Implementation

The component ships in `@clayui/core`:

```shell
npm install @clayui/css @clayui/core
```

```jsx
import {IconSelector} from '@clayui/core';
import {useState} from 'react';

export default function Example() {
	const [symbol, setSymbol] = useState('folder');

	return <IconSelector onSelectedIconChange={setSymbol} selectedIcon={symbol} />;
}
```

## Accessibility

- The trigger is a labeled button, and the icon grid is keyboard navigable with
  the arrow keys.
- Each icon option is named (the symbol name), so choices are announced, not
  just drawn.
- The search field filters without trapping focus.
