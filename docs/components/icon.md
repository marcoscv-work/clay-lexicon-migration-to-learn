# Icon

An icon is a visual representation of an idea or action, and by itself it has no
meaning: always associate it with a label or an action that gives it one. Choose
the icon and its label carefully so together they communicate clearly; if a
concept is hard to express with an icon, use text.

## Variants and anatomy

Lexicon icons have no intrinsic size: the container sets it, and the default is
16px. The icons live in the Liferay icon library (the Lexicon spritemap); do not
mix them with icons from other libraries. If the library lacks what you need,
switch libraries completely rather than mixing.

The library also includes language flags for the language selector. Flags are a
pragmatic choice: when the current language is unfamiliar, a flag stays
recognizable where a language name might not be readable.

Try the [Icon story](https://storybook.clayui.com/?path=/story/design-system-components-icon--default)
and the [context spritemap story](https://storybook.clayui.com/?path=/story/design-system-components-icon--context-spritemap).

## Usage guidelines

- Never let an icon carry meaning alone; the label does the communicating.
- Keep one icon library across the product.
- Size icons through their container, keeping the 16px default unless the
  context calls for another size.

## Implementation

Install the component and the CSS package:

```shell
npm install @clayui/css @clayui/icon
```

Reference a symbol from the spritemap; the `Provider` from `@clayui/core` can
supply the spritemap path once for the whole app:

```jsx
import Icon from '@clayui/icon';

export default function Example() {
	return <Icon spritemap="/icons.svg" symbol="folder" />;
}
```

## Accessibility

- Decorative icons (next to a text label) are hidden from assistive technology.
- An icon that stands alone in a control never does: the control needs an
  accessible name (see the icon button guidance on the
  [Button](/components/button) page).
- Do not convey state with icon color alone.
