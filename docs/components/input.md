# Input

Use a text input to let users enter data into the system. Every input has a
label, the field box, and optionally a placeholder, a status message, and help
text.

## Variants and anatomy

### States

An input responds with default, focused, disabled, and read only states, plus
three validation states: success, warning, and error. The validation message
sits right below the field, before the help text, so the user knows exactly
which field failed. Try the
[input feedback story](https://storybook.clayui.com/?path=/story/design-system-components-input--input-feedback).

### Sizes

Inputs come in small, regular, and large sizes; pick one and use it consistently
across a form.

### Text area

A text area accepts several lines of text and shares the input's states and
validation rules. The user can resize its height. When configured to grow with
the content, it starts at 44px, grows from its mid height (88px, three lines)
up to 200px, then scrolls. Try the
[textarea story](https://storybook.clayui.com/?path=/story/design-system-components-input--textarea).

### Input group

An input group attaches text, buttons, or other elements to a field, connected
or separated. Try the
[input group story](https://storybook.clayui.com/?path=/story/design-system-components-inputgroup--group-connected).

### Localized input

A localized input adds a language selector so users can provide a value per
language. Try the
[localized input story](https://storybook.clayui.com/?path=/story/design-system-components-localizedinput--default).

## Usage guidelines

- Prefer help text below the field. Use a help icon with a popover for on
  demand help.
- Avoid placeholders as help: users mistake them for values. If you use one,
  make it an example.
- Keep labels visible; do not replace the label with a placeholder.

## Implementation

The input ships in `@clayui/form`:

```shell
npm install @clayui/css @clayui/form
```

Wrap the field in a `ClayForm.Group` and add state classes for validation:

```jsx
import ClayForm, {ClayInput} from '@clayui/form';

export default function Example() {
	return (
		<ClayForm.Group className="has-error">
			<label htmlFor="email">Email</label>
			<ClayInput id="email" type="email" />
			<ClayForm.FeedbackGroup>
				<ClayForm.FeedbackItem>
					<ClayForm.FeedbackIndicator symbol="exclamation-full" />
					Enter a valid email address.
				</ClayForm.FeedbackItem>
			</ClayForm.FeedbackGroup>
		</ClayForm.Group>
	);
}
```

## Accessibility

- Associate the label with the field (`label` + `htmlFor`); never rely on the
  placeholder as the accessible name.
- Tie validation messages to the field with `aria-describedby`, and reflect the
  error state with `aria-invalid`.
- Keep the resize handle of text areas keyboard operable by letting content
  grow, not only drag.
