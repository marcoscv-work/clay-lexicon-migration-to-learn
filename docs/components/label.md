# Label

Use a label to categorize information with a short tag: to mark content, show a
status, filter, or link to related content. Labels provide quick recognition, so
keep them short and consistent.

## Variants and anatomy

### Simple labels

Simple labels identify, tag, or filter content. Status labels use the status
colors so users recognize meaning at a glance, in two weights:

- **Outline status labels**: the default. Use them where many status labels
  share the screen, such as tables and lists.
- **Bold status labels**: reserved for drawing attention to important
  information, for example one label in a header that must stand out. Their
  background and text keep a minimum contrast ratio of 5:1. Do not use bold as
  the default or next to many other status labels.

Try the color variants in the
[Label story](https://storybook.clayui.com/?path=/story/design-system-components-label--default).

### Removable label

Use a removable label in filter bars and category lists. It is always the
default size, has no status color, and shows a close icon to remove it. A
variation can include an icon or a round sticker before the text; try the
[content before story](https://storybook.clayui.com/?path=/story/design-system-components-label--content-before).

### Label with link

A label of any size can link to related content, for example a status label
that opens a modal with details, or a category label that opens a popover.

### Sizes

- **Small**: for labels that need no emphasis, always in uppercase; usual in
  tables, lists, and cards.
- **Default**: for marking states; lowercase is acceptable. Prefer it outside
  tables, for example in headers.

Try the [sizes story](https://storybook.clayui.com/?path=/story/design-system-components-label--sizes).

## Usage guidelines

- Keep the text short: one or two words.
- Keep labels to a single line, and never truncate them with an ellipsis. If
  space runs out, wrap the label to the next line.
- Do not place badges or other components inside a label.
- Use colored labels for status only; for plain categorization use the neutral
  variants.
- Do not use bold status labels alongside many other status labels.

## Implementation

Install the component and the CSS package:

```shell
npm install @clayui/css @clayui/label
```

Set the status color with `displayType`. By default the label is removable
(`withClose`); disable it for plain tags:

```jsx
import Label from '@clayui/label';

export default function Example() {
	return (
		<>
			<Label displayType="success" withClose={false}>
				Approved
			</Label>
			<Label displayType="warning" withClose={false}>
				Pending
			</Label>
			<Label onClose={() => {}}>Removable filter</Label>
		</>
	);
}
```

## Accessibility

- A label is informative text, not a control. When a label is removable, its
  close button needs an accessible name (for example `aria-label="Remove"`
  through `closeButtonProps`), so the action is announced.
- Bold status labels keep a 5:1 contrast ratio between text and background;
  keep that ratio if you customize colors.
- Do not rely on color alone: the label text itself must convey the status.
