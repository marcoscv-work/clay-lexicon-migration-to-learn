# Language Picker

Use the language picker to switch the working language of a field or a page.
Each locale shows its flag and name, following the icons foundation's rationale
for flags: they stay recognizable even when the current language's name is
unreadable to the user.

<!-- REVIEW: Clay-only component (no standalone Lexicon spec); it implements the language selector used by localized inputs. -->

## Variants and anatomy

The trigger shows the current locale; the panel lists the available locales. A
variant displays the translation status per language (translated, not
translated, default), which pairs with the
[localized input](/components/localized-input).

Try the [Language Picker story](https://storybook.clayui.com/?path=/story/design-system-components-languagepicker--default)
and the [with translations story](https://storybook.clayui.com/?path=/story/design-system-components-languagepicker--language-picker-with-translations).

## Usage guidelines

- Show the locale name next to the flag; the flag alone is ambiguous.
- Use the translation status variant in content editing contexts, so editors
  see what remains untranslated.
- Keep the list to the locales the content actually supports.

## Implementation

The component ships in `@clayui/core`:

```shell
npm install @clayui/css @clayui/core
```

```jsx
import {LanguagePicker} from '@clayui/core';

const locales = [
	{id: 'en_US', label: 'en-US', name: 'English (United States)', symbol: 'en-us'},
	{id: 'es_ES', label: 'es-ES', name: 'Spanish (Spain)', symbol: 'es-es'},
];

export default function Example() {
	return <LanguagePicker defaultSelectedLocaleId="en_US" locales={locales} />;
}
```

## Accessibility

- The trigger announces the current locale by name, not only its flag.
- The list is keyboard navigable, and each option's accessible name is the
  locale name.
- Translation status is conveyed in text (a label per option), not color alone.
