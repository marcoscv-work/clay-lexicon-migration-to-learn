# Link

Use a link, also known as a hyperlink, for navigation: it takes the user to
another place. A link never triggers any other kind of action; when you need an
action, use a [button](/components/button) instead (including the link-styled
button for low emphasis actions).

## Variants and anatomy

- **Inline link**: flows naturally inside a sentence or paragraph. It uses the
  primary color and underlines on hover.
- **Single link**: a standalone link, as text or an image. The primary text link
  uses the primary color in semibold; the secondary text link uses the secondary
  gray in semibold.
- **Title link**: for list item titles, such as rows in a table, asset titles,
  or blog entry teasers. It inherits the surrounding color and weight and
  underlines on hover.
- **External link**: points to a resource on another domain. Indicate the
  behavior with a tooltip, and provide an icon that is visually hidden by
  default but available on focus and to assistive technologies.

Try the display types in the
[Link story](https://storybook.clayui.com/?path=/story/design-system-components-link--default),
and the button-styled anchor in the
[displayed as button story](https://storybook.clayui.com/?path=/story/design-system-components-link--displayed-as-button).

## Usage guidelines

- Use links only for navigation. If activating the element runs an action, it is
  a button, even if it looks like a link.
- Make link text describe the destination; avoid generic labels such as "click
  here".
- Flag links that leave the site or open a new page, so the behavior is clear
  before activating them.

## Implementation

Install the component and the CSS package:

```shell
npm install @clayui/css @clayui/link
```

A Clay link renders as an anchor. Set the visual style with `displayType`:

```jsx
import Link from '@clayui/link';

export default function Example() {
	return (
		<Link displayType="primary" href="/documents">
			View all documents
		</Link>
	);
}
```

To customize how every link in your application renders (for example to
integrate a router), provide a component through `ClayLinkContext.Provider`.

## Accessibility

- A link's accessible name is its text; make it meaningful on its own, because
  screen reader users often navigate by listing links out of context.
- For external links, expose the "opens an external site" hint to assistive
  technology (Clay provides a visually hidden icon that appears on focus).
- Keep the visible focus indicator intact so keyboard users can track focus.
