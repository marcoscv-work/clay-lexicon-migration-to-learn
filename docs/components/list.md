# List

Use a list to present a dataset vertically when entries do not need exhaustive
comparison: it arranges data more flexibly than a [table](/components/table) and
is less visually explicit than [cards](/components/card). If entries need
exhaustive comparison, use the table.

## Variants and anatomy

Rows respond with default, hover, and selected states. A row can include a
checkbox or a radio (when paired with a management toolbar, always a checkbox),
a semibold main text, secondary descriptions, and actions.

**Sections** split the list into groups by category or type.

Try the [simple story](https://storybook.clayui.com/?path=/story/design-system-components-list--simple)
and the [complex story](https://storybook.clayui.com/?path=/story/design-system-components-list--complex).

## Usage guidelines

- Selection happens through the checkbox or radio, not the row background.
- One icon action can stand alone; two or more actions go in an actions menu on
  the right side of the row.
- Use the main text as an action only for a folder (navigate down) or a file
  (open its detail view). Preview is always a secondary action in the menu.
- Keep the main content under 50% of the row width on wide screens, stack at
  most three elements in it, and cap any text at two lines with an ellipsis.
- Tags in a row stay on one line, with an ellipsis when they overflow.

## Implementation

Install the component and the CSS package:

```shell
npm install @clayui/css @clayui/list
```

Compose rows from the list parts:

```jsx
import ClayList from '@clayui/list';

export default function Example() {
	return (
		<ClayList>
			<ClayList.Header>Documents</ClayList.Header>
			<ClayList.Item flex>
				<ClayList.ItemField expand>
					<ClayList.ItemTitle>Quarterly report</ClayList.ItemTitle>
					<ClayList.ItemText>Updated yesterday</ClayList.ItemText>
				</ClayList.ItemField>
			</ClayList.Item>
		</ClayList>
	);
}
```

## Accessibility

- The list renders semantic list markup, so structure is announced.
- Row actions (quick action menu) stay keyboard reachable, not hover-only.
- Selection checkboxes are labeled per row so each choice is identifiable.
