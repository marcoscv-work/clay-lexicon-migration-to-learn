# Loading Indicator

Use the loading indicator to tell the user that pending content will load where
the indicator appears, for example while data arrives from the server. Without
it, users can feel lost or leave the page prematurely.

## Variants and anatomy

### Shapes

- **Circle**: the default shape. Use it with small components such as badges,
  inputs, and table rows, in the `xs` and `sm` sizes with the `secondary` or
  `light` display type.
- **Squares**: two square shapes taken from the Liferay logo, adding a brand
  detail. Use it only for bigger surfaces such as cards, modals, sidebars, and
  dashboards, in the `md` and `lg` sizes with the `primary` or `light` display
  type.

### Sizes

Four sizes cover different contexts: small (10px), default (16px), medium
(32px), and large (64px). There is also a light variant for dark backgrounds.

Try it in the
[Loading Indicator story](https://storybook.clayui.com/?path=/story/design-system-components-loadingindicator--default).

## Usage guidelines

- Place the indicator where the pending content will appear.
- Do not put a loading indicator inside a button. If a button triggers a slow
  action, place the indicator next to the button instead.
- For processes with measurable progress, use a
  [progress bar](/components/progress-bar) instead.

## Implementation

Install the component and the CSS package:

```shell
npm install @clayui/css @clayui/loading-indicator
```

The indicator inherits `currentColor` and `font-size` by default. Choose the
shape, size, and display type for the context:

```jsx
import LoadingIndicator from '@clayui/loading-indicator';

export default function Example() {
	return (
		<>
			<LoadingIndicator displayType="secondary" size="sm" />
			<LoadingIndicator displayType="primary" shape="squares" size="md" />
		</>
	);
}
```

## Accessibility

- Announce loading to assistive technology: pair the indicator with text (even
  visually hidden) or a live region that reports the loading state, so the wait
  is not communicated visually only.
- Respect reduced motion preferences for long-running or large animations.
