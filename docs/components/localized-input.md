# Localized Input

Use a localized input when a field's value must exist per language, such as a
page title or a friendly URL. The field pairs a text input with a
[language picker](/components/language-picker) that switches the locale being
edited and shows each language's translation status.

## Variants and anatomy

The input behaves like a regular [input](/components/input) (same states,
sizes, and validation) with the locale selector attached. The selector marks
the default language and flags which locales still lack a translation.

Try the [Localized Input story](https://storybook.clayui.com/?path=/story/design-system-components-localizedinput--default).

## Usage guidelines

- Make the default language explicit; it is the fallback for missing
  translations.
- Show translation status so editors can see pending locales at a glance.
- Keep one localized value per field; do not multiply fields per language.

## Implementation

Install the component and the CSS package:

```shell
npm install @clayui/css @clayui/localized-input
```

Control the selected locale and the per-locale values:

```jsx
import ClayLocalizedInput from '@clayui/localized-input';
import {useState} from 'react';

const locales = [
	{label: 'en-US', symbol: 'en-us'},
	{label: 'es-ES', symbol: 'es-es'},
];

export default function Example() {
	const [selectedLocale, setSelectedLocale] = useState(locales[0]);
	const [translations, setTranslations] = useState({'en-US': 'Home'});

	return (
		<ClayLocalizedInput
			id="title"
			label="Title"
			locales={locales}
			onSelectedLocaleChange={setSelectedLocale}
			onTranslationsChange={setTranslations}
			selectedLocale={selectedLocale}
			translations={translations}
		/>
	);
}
```

## Accessibility

- The field keeps its label across locales; the selector announces the locale
  being edited.
- Translation status is text, announced with each locale option.
- Switching locale does not lose focus from the editing flow.
