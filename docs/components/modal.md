# Modal

Use a modal to communicate information or request an action without leaving the
current page. A modal temporarily blocks the rest of the workflow, so reserve it
for focused tasks that need the user's attention.

## Variants and anatomy

### Anatomy

![A modal with its parts labeled: header with title and close button, body, and footer with actions](/assets/components/modal-anatomy.png)

A modal has four parts: an optional header (title and close button), a required
body, an optional footer (actions), and an optional overlay behind it. Common
compositions are header with body and footer, header with body, and body with
footer.

### Sizes

![The modal sizes shown side by side: small, default, and large](/assets/components/modal-sizes.png)

Choose a size for the content: small (320px), default (600px), or large (896px).
A full screen size exists but is discouraged; prefer a dedicated page for large
tasks.

### Status types

![Status modals for information, success, warning, and error, each with an icon in the title](/assets/components/modal-status.png)

Besides the default modal, there are status modals for information, success,
warning, and error. A status modal must keep the icon in its title, because the
icon carries the meaning of the message.

## Usage guidelines

- Use a modal for simple, focused content. For complex or long forms, use a
  dedicated page instead.
- Do not open a modal from another modal.
- Do not let a modal cover the whole page; keep the context visible behind it.
- Align the footer buttons to the right. Order them left to right as borderless,
  secondary, then primary, and keep the confirming action (for example Save) on
  the right.
- Always give the user a way to cancel or close.
- Write button labels and the title as action verbs, so the purpose is clear.
- Keep the status icon in status modals; do not remove it.

## Implementation

Install the component and the CSS package:

```shell
npm install @clayui/css @clayui/modal
```

Control the modal with the `useModal` hook. It returns an `observer` to connect
the modal, an `onOpenChange` function to toggle it, and an `open` flag that gates
rendering:

```jsx
import Modal, {useModal} from '@clayui/modal';
import Button from '@clayui/button';

export default function Example() {
	const {observer, onOpenChange, open} = useModal();

	return (
		<>
			<Button onClick={() => onOpenChange(true)}>Open modal</Button>
			{open && (
				<Modal observer={observer} size="lg" status="info">
					<Modal.Header>Documents</Modal.Header>
					<Modal.Body>
						<p>Do you want to save your documents?</p>
					</Modal.Body>
					<Modal.Footer
						last={
							<Button.Group spaced>
								<Button displayType="secondary" onClick={() => onOpenChange(false)}>
									Cancel
								</Button>
								<Button onClick={() => onOpenChange(false)}>Save changes</Button>
							</Button.Group>
						}
					/>
				</Modal>
			)}
		</>
	);
}
```

## Accessibility

- Focus is trapped inside the modal while it is open, and returns to the element
  that opened it when it closes.
- Users can close the modal with `Esc`, the close button, or (when appropriate)
  the overlay. For a modal that requires a deliberate choice, do not allow
  dismissing it by clicking the overlay.
- `Tab` cycles through the focusable elements and wraps at the ends. Keep the
  close button labeled (for example `aria-label="Close"`).
- Keep the icon in status modals, because it conveys the message type to assistive
  technology as well as visually.
