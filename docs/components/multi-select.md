# Multi Select

Use Multi Select when the user builds a set of values by typing: each entry
becomes a tag rendered as a removable [label](/components/label). It is the
control for tagging, categorizing, and any multiple selection that grows from
free text.

## Variants and anatomy

Multi Select combines a text field with:

- **Autocomplete** of existing values while typing.
- **Fast tag creation on paste**: pasted text separated by commas becomes one
  tag per element.
- **A character allowlist**, so tags only accept the characters you configure.

The field grows to several lines as tags accumulate. Try the
[Multi Select story](https://storybook.clayui.com/?path=/story/design-system-components-multiselect--default)
and the [primary action story](https://storybook.clayui.com/?path=/story/design-system-components-multiselect--with-primary-action).

## Usage guidelines

- Use Multi Select for open sets the user extends by typing. For choosing among
  a fixed short list, use a [select](/components/select) or checkboxes.
- Keep tags short; they render as labels and follow label rules (one line, no
  ellipsis).
- Offer autocomplete whenever existing values are likely to be reused, to avoid
  near duplicate tags.

## Implementation

Install the component and the CSS package:

```shell
npm install @clayui/css @clayui/multi-select
```

Control the field value and the tag list; `sourceItems` feeds the autocomplete:

```jsx
import ClayMultiSelect from '@clayui/multi-select';
import {useState} from 'react';

const sourceItems = [
	{label: 'News', value: 'news'},
	{label: 'Events', value: 'events'},
];

export default function Example() {
	const [value, setValue] = useState('');
	const [items, setItems] = useState([]);

	return (
		<ClayMultiSelect
			inputName="tags"
			items={items}
			onChange={setValue}
			onItemsChange={setItems}
			sourceItems={sourceItems}
			value={value}
		/>
	);
}
```

## Accessibility

- The field needs a label like any input; associate it with the control.
- Each tag's remove button carries an accessible name so removal is announced.
- Autocomplete suggestions are keyboard navigable (arrow keys, `Enter` to
  accept).
