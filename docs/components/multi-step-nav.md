# Multi Step Nav

Use the multi step nav (the multi step form pattern) to divide a long or
complicated task into smaller steps, giving the user room to breathe and a sense
of progression. Use it when the process has a required sequence or guides the
user through something unfamiliar.

For system progress (a file upload), use a
[progress bar](/components/progress-bar) instead.

## Variants and anatomy

Each step has three states: **done** (checked), **selected** (current), and
**disabled** (not visited). The steps row divides the space equally.

- A multi step form needs at least two steps.
- Prefer at most seven steps: easy to remember and comfortable on touch
  devices. Beyond seven, group the remaining steps in a drop down under the
  sixth element.
- On mobile, condense to five steps at most, with the rest in a drop down under
  the second to last step.

Try the [Multi Step Nav story](https://storybook.clayui.com/?path=/story/design-system-components-multistepnav--default)
and the [error state story](https://storybook.clayui.com/?path=/story/design-system-components-multistepnav--error).

## Usage guidelines

- Make each chunk manageable; do not overwhelm a step.
- Name steps with clear, short titles: two words maximum.
- When the process allows going back, make the completed steps links.

## Implementation

Install the component and the CSS package:

```shell
npm install @clayui/css @clayui/multi-step-nav
```

The high level component renders the steps from an array:

```jsx
import {ClayMultiStepNavWithBasicItems} from '@clayui/multi-step-nav';
import {useState} from 'react';

export default function Example() {
	const [value, setValue] = useState(1);

	return (
		<ClayMultiStepNavWithBasicItems
			active={value}
			defaultOnClick={(_event, index) => setValue(index)}
			items={[
				{subTitle: 'Details', title: '1'},
				{subTitle: 'Options', title: '2'},
				{subTitle: 'Review', title: '3'},
			]}
			maxStepsShown={7}
		/>
	);
}
```

## Accessibility

- Steps expose their state (current, completed, disabled) to assistive
  technology; the current step carries `aria-current="step"`.
- Steps that allow navigation are real links or buttons, keyboard operable.
- The overflow drop down keeps collapsed steps reachable.
