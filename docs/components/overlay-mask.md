# Overlay Mask

Use the overlay mask to dim the page except for a highlighted element, focusing
the user's attention on it. Its typical use is onboarding and guided tours,
often paired with a [popover](/components/popover) explaining the highlighted
element.

<!-- REVIEW: Clay-only component (no Lexicon spec). -->

## Variants and anatomy

The mask wraps the target area and dims everything else with configurable
padding around the highlight. Combine it with a popover to build a step of a
tour; try the
[Overlay Mask story](https://storybook.clayui.com/?path=/story/design-system-components-overlaymask--default)
and the [with popover story](https://storybook.clayui.com/?path=/story/design-system-components-overlaymask--with-popover).

## Usage guidelines

- Use it sparingly: a mask interrupts everything else on the page.
- Always pair the highlight with an explanation and a clear way to advance or
  dismiss the tour.
- Keep the highlighted element interactive if the step asks the user to try it.

## Implementation

The component ships in `@clayui/core`:

```shell
npm install @clayui/css @clayui/core
```

```jsx
import {OverlayMask} from '@clayui/core';
import Button from '@clayui/button';
import {useState} from 'react';

export default function Example() {
	const [visible, setVisible] = useState(true);

	return (
		<OverlayMask visible={visible}>
			<Button onClick={() => setVisible(!visible)}>Try this action</Button>
		</OverlayMask>
	);
}
```

## Accessibility

- The dimmed content remains in the DOM; make sure the tour moves focus to the
  highlighted element or its explanation.
- Provide a keyboard way to dismiss or advance (`Esc` and a visible button).
- Do not rely on dimming alone: name the highlighted element in the
  explanation.
