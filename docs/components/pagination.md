# Pagination

Use pagination to navigate horizontally between chunks (pages) of a dataset. Use
the plain pagination for a fixed page size, and the pagination bar when the user
also needs to change the page size.

## Variants and anatomy

### Pagination

The default pattern combines four mechanisms:

- **Left arrow**: goes back one page; disabled on the first page.
- **Numbers**: jump straight to a page.
- **Ellipsis**: appears when there are more than six pages, always in a middle
  position, and opens a drop down with the remaining pages.
- **Right arrow**: goes forward one page; disabled on the last page.

Try the [Pagination story](https://storybook.clayui.com/?path=/story/design-system-components-pagination--default)
and the [disabled pages story](https://storybook.clayui.com/?path=/story/design-system-components-pagination--disabled-pages).

### Pagination bar

The pagination bar adds, from left to right: a page size selector, a label
describing the exact segment shown ("Showing 1 to 20 of 43"), and the classic
pagination. Use it when changing the page size must recalculate the number of
pages.

Try the [Pagination Bar story](https://storybook.clayui.com/?path=/story/design-system-components-paginationbar--default).

### Pager

For chunked processes such as long forms, a simple previous and next pager (no
page numbers) can be enough. Prefer full pagination for datasets.

## Usage guidelines

- Keep pagination attached to the dataset it controls, usually at the bottom.
- Disable the arrows at the limits instead of hiding them, so the layout stays
  stable.
- Offer sensible page size options in the bar and remember the user's choice
  where possible.

## Implementation

Install the components and the CSS package:

```shell
npm install @clayui/css @clayui/pagination @clayui/pagination-bar
```

The high level component renders the full pattern from the total number of
items:

```jsx
import {ClayPaginationWithBasicItems} from '@clayui/pagination';

export default function Example({page, setPage}) {
	return (
		<ClayPaginationWithBasicItems
			active={page}
			ellipsisBuffer={2}
			onActiveChange={setPage}
			totalPages={25}
		/>
	);
}
```

For the bar, `ClayPaginationBarWithBasicItems` combines the page size drop down,
the results label, and the pagination in one component driven by `activeDelta`,
`totalItems`, and `onDeltaChange`.

## Accessibility

- Pagination renders in a navigation landmark with the current page marked, so
  the position is announced.
- Arrows at their limit are disabled, not removed, and stay out of the tab
  order confusion.
- The ellipsis drop down keeps the intermediate pages keyboard reachable.
