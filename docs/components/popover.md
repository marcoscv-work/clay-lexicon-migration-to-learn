# Popover

Use a popover for contextual help that does not fit a tooltip: an explanation of
a setting, extra detail about a concept, or supporting content with a title.
When the content is a short hint of a few words, use a
[tooltip](/components/tooltip) instead; a good rule of thumb is that text beyond
about 280 characters belongs in a popover.

## Variants and anatomy

A popover has an arrow pointing at the element it describes, an optional header,
and a body. It can open on any side of its trigger; align the arrow so it
clearly points at the right element.

A common pattern pairs the popover with a help icon (a question mark) next to
the term it explains: place it 8px from body text or 16px from a section title.

Try the [Popover story](https://storybook.clayui.com/?path=/story/design-system-components-popover--default)
and the [trigger story](https://storybook.clayui.com/?path=/story/design-system-components-popover--trigger),
which opens the popover from a button.

## Usage guidelines

- Keep the content focused: a short header and a paragraph. If the user needs a
  form or a workflow, use a [modal](/components/modal) or a panel instead.
- Position the popover so it does not cover the element it explains or other
  interactive elements.
- Let users dismiss it easily: clicking outside or pressing `Esc`.

## Implementation

Install the component and the CSS package:

```shell
npm install @clayui/css @clayui/popover
```

Render a popover with a header and an alignment, or attach it to a trigger so it
opens on interaction:

```jsx
import Popover from '@clayui/popover';
import Button from '@clayui/button';

export default function Example() {
	return (
		<Popover
			alignPosition="bottom"
			header="What is a vocabulary?"
			trigger={<Button displayType="secondary">More info</Button>}
		>
			A vocabulary groups related categories so you can organize your
			content consistently.
		</Popover>
	);
}
```

## Accessibility

- Make the trigger a real control (a button) with an accessible name, so the
  popover is reachable and operable by keyboard.
- The popover content must be dismissible without moving the pointer (`Esc`)
  and must not trap focus.
- Do not put essential information only in a hover-triggered popover; hover
  does not exist on touch devices.
