# Progress Bar

Use a progress bar as feedback for a running system task, such as a file upload.
While the process runs, the bar grows continuously from 0% to 100%.

Never use a progress bar to communicate a user's progress through a set of
steps; that is what a multi step form is for.

## Variants and anatomy

The bar has three states:

- **Loading**: the bar grows in the primary color and the label shows the
  completion percentage.
- **Warning**: something interrupted the process.
- **Finished**: the process completed successfully; the label changes to a
  success icon.

Try the states in the
[Progress Bar story](https://storybook.clayui.com/?path=/story/design-system-components-progressbar--default).

## Usage guidelines

- Use it for system processes with measurable progress, such as uploads.
- Keep the percentage label visible so the user knows how much remains.
- When the process fails or pauses, switch to the warning state instead of
  leaving the bar frozen.

## Implementation

Install the component and the CSS package:

```shell
npm install @clayui/css @clayui/progress-bar
```

Set the completion with `value` (0 to 100). At 100 the bar styles itself as
success; use `warn` to fix it in the warning color:

```jsx
import ProgressBar from '@clayui/progress-bar';

export default function Example({percent}) {
	return (
		<>
			<ProgressBar value={percent} />
			<ProgressBar value={45} warn />
		</>
	);
}
```

## Accessibility

- The progress bar exposes its state through the ARIA progressbar pattern
  (current, minimum, and maximum values), so assistive technology can announce
  progress.
- Do not communicate the warning state with color alone; the label change makes
  the state readable without color.
