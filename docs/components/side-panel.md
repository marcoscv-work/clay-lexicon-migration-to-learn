# Side Panel

Use a side panel (sidebar) for vertical panels that appear from the left or
right side of the screen, triggered by a button. Lexicon defines two canonical
uses: the **product menu** (navigation between applications) and the **info
panel** (details that do not fit the table, list, or card view).

## Variants and anatomy

On desktop the panel pushes the content aside; on smaller viewports it slides
over the content instead (below 768px for the product menu, below 992px for the
info panel).

Try the [absolute position story](https://storybook.clayui.com/?path=/story/design-system-components-sidepanel--position-absolute),
the [fixed position story](https://storybook.clayui.com/?path=/story/design-system-components-sidepanel--position-fixed),
and the [drilldown story](https://storybook.clayui.com/?path=/story/design-system-components-sidepanel--drilldown)
for multi level panel content.

## Usage guidelines

- Trigger the panel at its correct hierarchical level: the product menu at
  application level, the info panel next to the dataset it describes.
- Keep one clear way to close the panel, and return focus to the trigger.
- Use the info panel for supporting detail, not for primary workflows; those
  belong on a page or a [modal](/components/modal).

## Implementation

The component ships in `@clayui/core`:

```shell
npm install @clayui/css @clayui/core
```

Bind the panel to its trigger with an id and control the open state:

```jsx
import {SidePanel} from '@clayui/core';
import Button from '@clayui/button';
import {useState} from 'react';

export default function Example() {
	const [open, setOpen] = useState(false);

	return (
		<>
			<Button aria-controls="infoPanel" onClick={() => setOpen(!open)}>
				Info
			</Button>
			<SidePanel
				aria-label="Item details"
				id="infoPanel"
				onOpenChange={setOpen}
				open={open}
			>
				<SidePanel.Header>Details</SidePanel.Header>
				<SidePanel.Body>Contextual information here.</SidePanel.Body>
			</SidePanel>
		</>
	);
}
```

## Accessibility

- The panel is a labeled region; name it with `aria-label` or a visible header.
- Opening moves focus into the panel; closing (including with `Esc`) returns
  focus to the trigger.
- When the panel overlays content on small screens, background content is not
  reachable while it is open.
