# Slider

Use a slider when the user should play with a value inside a linear range:
image contrast, audio volume, a video position, or a filter in a faceted
search. The default range is 0 to 100, and both limits are configurable.

## Variants and anatomy

Three parts form the slider: the **track** (the full range), the **thumb** (the
knob that selects the value), and the **progress** (the filled part up to the
value). States: active, hover, disabled, and focused. An optional tooltip shows
the current value on hover over the thumb.

A **step** defines the size of each thumb movement; clicking the track moves
the thumb to that point (snapped to the step when one is set).

Try the [Slider story](https://storybook.clayui.com/?path=/story/design-system-components-slider--default).

## Usage guidelines

- Use a slider when exploring values matters more than typing an exact number;
  pair it with an input when precision is also needed.
- Set a step that matches the meaning of the value.
- Show the current value (tooltip or companion label) at all times during
  interaction.

## Implementation

Install the component and the CSS package:

```shell
npm install @clayui/css @clayui/slider
```

Control the value:

```jsx
import ClaySlider from '@clayui/slider';
import {useState} from 'react';

export default function Example() {
	const [value, setValue] = useState(50);

	return <ClaySlider max={100} min={0} onChange={setValue} step={5} value={value} />;
}
```

## Accessibility

- The slider follows the ARIA slider pattern with full keyboard support: arrow
  keys move one step, `Page Up` and `Page Down` move a larger amount, and
  `Home` and `End` jump to the limits.
- Label the slider with the value it controls, and expose the current value
  (`aria-valuenow` comes from the component).
- Do not make the tooltip the only way to read the value.
