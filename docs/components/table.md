# Table

Use a table to compare datasets in a direct, analytical, text oriented way. When
the data is image heavy, use [cards](/components/card) or a list instead.

## Variants and anatomy

### Anatomy

![A table with its parts labeled: header row, group section, data rows, and row actions](/assets/components/table-anatomy.jpg)

A table has a header, optional sections that group rows, and data rows. A row can
include a selection checkbox, a main column, other columns, and an actions button.
On hover, a row can reveal up to three quick action icons.

### Row states

![Table rows in their default, hover, and selected states](/assets/components/table-row-states.jpg)

Rows have default, hover, and selected states. The hover state reveals quick
actions, which are hidden while a row is selected. The header supports sorting
(an icon shows the direction) and column drag and drop.

### Content

You can render a table with static content (rows written directly) or dynamic
content (a render function over a list of items), which keeps the table data
agnostic and supports features such as sorting, nested rows, and lazy loading.

## Usage guidelines

- Keep one type of data per column; do not mix two data types in a single column.
- Put the main column first and set it in semibold. Left align headers and set
  them in semibold. Right align numeric values.
- Use a selection checkbox when the table works with a management toolbar.
- Make the main column text a navigation action only when it makes sense, for
  example to open a folder or a file's detail view.
- Put multiple row actions in an actions menu on the right. A single icon action
  can stand on its own.
- Do not place a static button in a row's default state, because the hover quick
  actions would overlap it.
- Visually flag a link that leaves the site or opens a new page.

## Implementation

The table ships in `@clayui/core`. Install it and the CSS package:

```shell
npm install @clayui/css @clayui/core
```

Build a static table from the `Table`, `Head`, `Body`, `Row`, and `Cell` parts:

```jsx
import {Body, Cell, Head, Row, Table} from '@clayui/core';

export default function Example() {
	return (
		<Table>
			<Head>
				<Cell key="name">Name</Cell>
				<Cell key="type">Type</Cell>
			</Head>
			<Body>
				<Row>
					<Cell>Games</Cell>
					<Cell>File folder</Cell>
				</Row>
				<Row>
					<Cell>Program files</Cell>
					<Cell>File folder</Cell>
				</Row>
			</Body>
		</Table>
	);
}
```

For larger datasets, pass `items` to `Head` and `Body` and render each column and
row with a function. Icons in the table require an application `Provider` with a
`spritemap`.

## Accessibility

- Give the `Table` an `aria-label` so its purpose is announced.
- The table follows the W3C ARIA table pattern by default and switches to the
  treegrid pattern automatically when you use nested rows.
- For nested rows, give each row's data an `id` (or configure `itemIdKey`), so
  expansion is tracked correctly.
