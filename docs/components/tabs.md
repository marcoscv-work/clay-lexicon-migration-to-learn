# Tabs

Use tabs to move between views of related content in the same context, splitting
information into groups the user can identify easily. Use tabs when the user
does not need to compare the contents side by side.

## Variants and anatomy

- **Classic tab**: use inside forms.
- **Modern tab**: use with dataset visualizations, such as list or card views.
  It drops the background and underlines the selected tab.

All tabs in a bar must share the same style and type; mixing styles reads as
navigation, which is not what tabs are for. When tabs do not fit on the screen,
collapse the remaining ones under a final "More" drop down tab. If the bar needs
visual context, a thin line under it helps.

Try both styles in the
[Tabs story](https://storybook.clayui.com/?path=/story/design-system-components-tabs--default)
and the [modern style story](https://storybook.clayui.com/?path=/story/design-system-components-tabs--new-default).

## Usage guidelines

- Keep one tab selected at all times, and pre-select one by default. The
  selected tab must clearly stand out, and only one can be selected at a time.
- Order the tabs in a way that is natural for your users.
- Keep labels short, clear, and concise: two words at most.
- Do not set labels in all caps; it hurts readability.

## Implementation

Install the component and the CSS package:

```shell
npm install @clayui/css @clayui/tabs
```

Tabs is a controlled component: you hold the active index in state. Compose the
triggers with `Tabs.Item` and the panels with `Tabs.Content` and `Tabs.TabPane`:

```jsx
import Tabs from '@clayui/tabs';
import {useState} from 'react';

export default function Example() {
	const [active, setActive] = useState(0);

	return (
		<>
			<Tabs active={active} onActiveChange={setActive}>
				<Tabs.Item innerProps={{'aria-controls': 'panel-1'}}>Details</Tabs.Item>
				<Tabs.Item innerProps={{'aria-controls': 'panel-2'}}>History</Tabs.Item>
			</Tabs>
			<Tabs.Content activeIndex={active} fade>
				<Tabs.TabPane aria-labelledby="tab-1">Details content.</Tabs.TabPane>
				<Tabs.TabPane aria-labelledby="tab-2">History content.</Tabs.TabPane>
			</Tabs.Content>
		</>
	);
}
```

Give a `Tabs.Item` an `href` to render it as an anchor instead of a button.

## Accessibility

- Wire each trigger to its panel: `aria-controls` on the `Tabs.Item` and
  `aria-labelledby` on the `Tabs.TabPane`, so the relationship is announced.
- Tabs follow the ARIA tabs pattern: arrow keys move between tabs and the
  selected tab is exposed with `aria-selected`.
- Keep tab labels text based so they are readable by assistive technology.
