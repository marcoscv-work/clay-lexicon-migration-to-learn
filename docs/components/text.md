# Text

Use the text component for body copy that follows the
[typography foundation](/foundations/typography): the scale sizes, the three
weights, and the semantic colors (secondary, muted, and status colors), without
writing custom CSS.

## Variants and anatomy

`size` maps to the type scale, `weight` to the foundation's three weights, and
`color` to the semantic text colors. The companion `TextHighlight` marks
fragments, for example matched characters in an
[autocomplete](/components/autocomplete).

Try the [text typography story](https://storybook.clayui.com/?path=/story/design-system-components-typography--text-typography)
and the [text highlight story](https://storybook.clayui.com/?path=/story/design-system-components-typography--text-highlight-typography).

## Usage guidelines

- Prefer the semantic colors (secondary for supporting text, status colors for
  meaning) over custom ones, so themes keep working.
- Keep body text at the base size with the standard 1.5 line height; reserve
  smaller sizes for supporting text.

## Implementation

The component ships in `@clayui/core`:

```shell
npm install @clayui/css @clayui/core
```

```jsx
import {Text} from '@clayui/core';

export default function Example() {
	return (
		<Text color="secondary" size={3}>
			Updated yesterday at 16:32
		</Text>
	);
}
```

## Accessibility

- Semantic colors keep the foundation's contrast ratios; check contrast again
  if you override them.
- Do not convey meaning with color alone; status text still names the status.
