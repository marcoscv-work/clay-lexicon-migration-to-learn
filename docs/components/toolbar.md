# Toolbar

Use a toolbar to group the actions of a specific context into a horizontal bar.
The bar is responsive, grows with the height of its content, and keeps its
elements vertically aligned.

## Variants and anatomy

You build a toolbar by declaring blocks and placing elements inside them. There
are two block types:

- **Field**: a tight group, for items such as action icons.
- **Content**: fills the remaining space; several content blocks split the free
  space equally.

Common layouts combine them: three fields, three contents, or field, content,
field. Horizontal distance between elements is 16px (8px on mobile).

Try the [Toolbar story](https://storybook.clayui.com/?path=/story/design-system-components-toolbar--default)
and the [upper toolbar story](https://storybook.clayui.com/?path=/story/design-system-components-toolbar--upper-toolbar),
the variant used at the top of edit screens.

For dataset views (search, filters, bulk selection), use the
[Management Toolbar](/components/management-toolbar), which is built with these
same blocks.

## Usage guidelines

- Group related actions in the same field block, and put the primary action
  where the reading flow ends.
- Keep vertical spacing at 16px minimum; a search field (40px tall) reduces the
  bar's vertical margins to 12px to keep its emphasis.
- Do not overload the bar; overflow secondary actions into a
  [drop down](/components/drop-down).

## Implementation

Install the component and the CSS package:

```shell
npm install @clayui/css @clayui/toolbar
```

Compose the bar from rows, items, and sections:

```jsx
import Toolbar from '@clayui/toolbar';
import Button from '@clayui/button';

export default function Example() {
	return (
		<Toolbar>
			<Toolbar.Nav>
				<Toolbar.Item expand>
					<Toolbar.Section>
						<label className="component-title">Entries</label>
					</Toolbar.Section>
				</Toolbar.Item>
				<Toolbar.Item>
					<Button displayType="primary">New Entry</Button>
				</Toolbar.Item>
			</Toolbar.Nav>
		</Toolbar>
	);
}
```

## Accessibility

- Give the toolbar an accessible name (`aria-label`) describing its context.
- Icon-only actions inside the bar need accessible names, and tooltips help
  sighted users identify them.
- Keep the tab order following the visual order of the blocks.
