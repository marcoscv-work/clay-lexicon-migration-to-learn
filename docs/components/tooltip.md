# Tooltip

Use a tooltip to show a brief text label on hover that clarifies an element's
meaning or use. Tooltips are for short, supplementary hints. When the content is
longer, use a popover instead.

## Variants and anatomy

A tooltip appears when the pointer hovers over an element and disappears on mouse
out. Because it is hover only, it does not exist on touch devices, so never put
essential information only in a tooltip.

The arrow points at the element the tooltip describes. Clay supports twelve arrow
positions, so the tooltip can sit above, below, left, or right of the element,
aligned to its start, center, or end. Try the
[Tooltip story](https://storybook.clayui.com/?path=/story/design-system-components-tooltip--default)
and switch the alignment in the controls, or the
[provider story](https://storybook.clayui.com/?path=/story/design-system-components-tooltip--tooltip-provider)
to see tooltips driven by `title` attributes.

## Usage guidelines

- Use a tooltip on elements that trigger an action, such as an icon button. You
  can also use one on a link or an image.
- Keep the text brief. Aim for about 280 characters or fewer; if you need more,
  use a popover.
- Position the arrow so it clearly points at the correct element, and avoid
  overlapping other interactive elements.
- Do not put tooltips on plain body text.

## Implementation

Install the component and the CSS package:

```shell
npm install @clayui/css @clayui/tooltip
```

Render a tooltip directly with an alignment and content:

```jsx
import Tooltip from '@clayui/tooltip';

export default function Example() {
	return (
		<Tooltip alignPosition="bottom" show>
			Tooltip content
		</Tooltip>
	);
}
```

More often, wrap a region in `ClayTooltipProvider` once, then annotate any child
element with a `title` attribute and an optional `data-tooltip-align`:

```jsx
import {ClayTooltipProvider} from '@clayui/tooltip';

export default function Example() {
	return (
		<ClayTooltipProvider>
			<button data-tooltip-align="top" title="Add template">
				<span aria-hidden="true">+</span>
			</button>
		</ClayTooltipProvider>
	);
}
```

## Accessibility

- Do not rely on a tooltip as the only source of important information, since it
  is not available on touch devices and appears only on hover.
- For a disabled control that still needs a tooltip, the native `disabled`
  attribute suppresses the events the provider listens for, so the tooltip will
  not appear. Instead, keep the element active but mark it with a `disabled` CSS
  class and `aria-disabled="true"`.
- Make sure an icon only control that uses a tooltip also has an accessible name
  (for example through `aria-label`), because the tooltip text is not guaranteed
  to be read by assistive technology.
