# Tree View

Use a tree view to display information in a hierarchical structure of
collapsible nodes: the classic example is a folder tree, but it fits any
hierarchical relationship. Users navigate it with the mouse or entirely with the
keyboard.

## Variants and anatomy

A tree view is a box that constrains its **nodes**. Each node stacks, left to
right: the expander, an optional checkbox (with tri-state support), one icon
before the text for the item's type, the text, optional state icons after it,
and a "more actions" area on the right.

Node roles follow the hierarchy: root, child, parent (expandable or collapsed),
and end nodes (no children). Light and dark schemes are available; try the
[dynamic story](https://storybook.clayui.com/?path=/story/design-system-components-treeview--dynamic)
and the [dark theme story](https://storybook.clayui.com/?path=/story/design-system-components-treeview--theme-dark).

Common variants: nodes with user stickers, adjusted text sizes and density, and
toggle button selection; see the
[sticker story](https://storybook.clayui.com/?path=/story/design-system-components-treeview--with-sticker)
and the [multiple selection story](https://storybook.clayui.com/?path=/story/design-system-components-treeview--multiple-selection).

## Usage guidelines

- With multiple selection, use tri-state checkboxes for hierarchies: checking a
  parent checks all its children; a partially selected parent shows the
  indeterminate state.
- Reveal per-node actions on hover (and directly on selected nodes).
- For very large trees, load children asynchronously; see the
  [async load story](https://storybook.clayui.com/?path=/story/design-system-components-treeview--async-load).

## Implementation

The component ships in `@clayui/core`:

```shell
npm install @clayui/css @clayui/core
```

Provide the hierarchy as `items` (nested through a key you choose) and render
each node:

```jsx
import {TreeView} from '@clayui/core';

const items = [
	{
		children: [{id: 2, name: 'Reports'}],
		id: 1,
		name: 'Documents',
	},
];

export default function Example() {
	return (
		<TreeView items={items} nestedKey="children">
			{(item) => (
				<TreeView.Item key={item.id}>
					<TreeView.ItemStack>{item.name}</TreeView.ItemStack>
					{(child) => <TreeView.Item key={child.id}>{child.name}</TreeView.Item>}
				</TreeView.Item>
			)}
		</TreeView>
	);
}
```

## Accessibility

- The component implements the ARIA tree pattern. Once focus enters the tree,
  the arrow keys move between nodes (up and down) and collapse or expand them
  (left and right); `Tab` leaves the tree.
- Disabled nodes do not receive focus.
- Everything achievable with the mouse (selection, expansion, actions) is
  achievable with the keyboard.
