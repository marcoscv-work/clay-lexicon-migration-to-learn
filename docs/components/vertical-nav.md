# Vertical Nav

Use vertical navigation, placed on the left side of the content, to move between
pages or forms in lower navigation levels. It is common in form templates: split
a long form into smaller forms and let the vertical nav move between them.

## Variants and anatomy

Items respond with default, hover, active, and selected states, and can nest to
represent hierarchy, expanding and collapsing their children.

On mobile, the navigation becomes a full width collapsible menu displayed before
the form. The open menu overlays the form; it never pushes the content down.

Try the [Vertical Nav story](https://storybook.clayui.com/?path=/story/design-system-components-verticalnav--default)
and the [primary variant story](https://storybook.clayui.com/?path=/story/design-system-components-verticalnav--primary).

## Usage guidelines

- Use vertical navigation for lower levels of navigation, not as the top level
  of an application.
- Split long forms into smaller pages connected by the vertical nav, so each
  task stays manageable.
- Keep one item marked as active so the user always knows where they are.

## Implementation

The component ships in `@clayui/core`. Install it and the CSS package:

```shell
npm install @clayui/css @clayui/core
```

Pass the tree as `items` and render each one; nested arrays create expandable
levels:

```jsx
import {VerticalNav} from '@clayui/core';

const items = [
	{
		id: 1,
		label: 'Site Settings',
		items: [
			{href: '/general', id: 2, label: 'General'},
			{href: '/seo', id: 3, label: 'SEO'},
		],
	},
];

export default function Example() {
	return (
		<VerticalNav defaultExpandedKeys={new Set([1])} items={items}>
			{(item) => (
				<VerticalNav.Item href={item.href} items={item.items} key={item.id}>
					{item.label}
				</VerticalNav.Item>
			)}
		</VerticalNav>
	);
}
```

## Accessibility

- The component renders a navigation landmark; name it with `aria-label` when
  there are several navs on the page.
- Expansion is keyboard operable and exposed with `aria-expanded` on parent
  items; the current item should carry `aria-current="page"`.
- On mobile, the collapsible trigger is a labeled button, and the overlay menu
  stays keyboard reachable.
